Layer 4: Transport Layer

Reliable end-to-end delivery

What is the Transport Layer?

The Transport Layer is responsible for end-to-end communication and data integrity. It ensures complete data transfer between two hosts, handling segmentation, flow control, and error correction.

This layer introduces the concept of ports - which allow multiple applications to use the network simultaneously.

Key Functions

  • Segmentation: Breaks data into smaller segments
  • Flow Control: Manages data transmission rate
  • Error Control: Detects and retransmits lost segments
  • Port Addressing: Uses ports to identify applications
  • Connection Management: Establishes and terminates connections

TCP vs UDP

  • TCP (Transmission Control Protocol):
    • Connection-oriented (3-way handshake)
    • Reliable - guarantees delivery
    • Ordered - data arrives in sequence
    • Slower due to overhead
    • Used for: Web, Email, File Transfer
  • UDP (User Datagram Protocol):
    • Connectionless
    • Unreliable - no delivery guarantee
    • Faster - minimal overhead
    • Used for: Streaming, Gaming, DNS, VoIP

Common Port Numbers

  • Port 80 - HTTP (Web)
  • Port 443 - HTTPS (Secure Web)
  • Port 21 - FTP
  • Port 22 - SSH
  • Port 25 - SMTP (Email)
  • Port 53 - DNS

Real-World Example

TCP Three-Way Handshake:

1. SYN - Client: "Hey, can we talk?"

2. SYN-ACK - Server: "Sure, I'm ready!"

3. ACK - Client: "Great, let's begin!"
