Layer 3: Network Layer

The routing and addressing layer

What is the Network Layer?

The Network Layer is responsible for logical addressing (IP addresses) and routing data packets from source to destination across multiple networks.

This is where routers operate - they read IP addresses and determine the best path for data to travel.

Key Functions

  • Logical Addressing: Assigns IP addresses to identify devices
  • Routing: Determines the best path for packets
  • Packet Forwarding: Moves packets hop-by-hop to destination
  • Fragmentation: Breaks packets if they're too large
  • Error Handling: Reports delivery problems (ICMP)

IP Addressing

  • IPv4: 32-bit address (e.g., 192.168.1.1)
    • ~4.3 billion unique addresses
    • Format: xxx.xxx.xxx.xxx (0-255 each)
  • IPv6: 128-bit address (e.g., 2001:0db8:85a3::8a2e:0370:7334)
    • 340 undecillion unique addresses
    • Designed to replace IPv4

Common Protocols & Devices

  • IP - Internet Protocol
  • ICMP - Internet Control Message Protocol (ping)
  • ARP - Address Resolution Protocol
  • OSPF, BGP, RIP - Routing protocols
  • Routers - Layer 3 devices

Real-World Example

When you visit a website:

1. Your device has IP: 192.168.1.100

2. Website server has IP: 142.250.190.78 (Google)

3. Your router checks routing table

4. Packet travels through multiple routers

5. Each router forwards based on IP destination
