Layer 2: Data Link Layer

Node-to-node data transfer

What is the Data Link Layer?

The Data Link Layer is responsible for node-to-node delivery of data. It packages bits from the Physical layer into frames and handles physical addressing using MAC addresses.

This layer operates within a local network (LAN) and is where switches operate.

Key Functions

  • Framing: Packages data into frames with headers/trailers
  • Physical Addressing: Uses MAC addresses for local delivery
  • Error Detection: Uses CRC to detect transmission errors
  • Flow Control: Manages frame transmission rate
  • Access Control: Determines which device can transmit

Two Sub-layers

  • LLC (Logical Link Control):
    • Interfaces with the Network layer
    • Flow control and error management
  • MAC (Media Access Control):
    • Interfaces with the Physical layer
    • Handles MAC addressing
    • Controls access to shared medium

MAC Address

A MAC address is a 48-bit hardware address burned into every network interface card (NIC).

Format: AA:BB:CC:DD:EE:FF

First 3 bytes = Manufacturer (OUI)

Last 3 bytes = Device identifier

Example: 00:1A:2B:3C:4D:5E

Common Protocols & Devices

  • Ethernet - Most common LAN technology
  • Wi-Fi (802.11) - Wireless LAN
  • PPP - Point-to-Point Protocol
  • Switches - Layer 2 devices
  • Bridges - Connect network segments

Real-World Example

When data is sent on your local network:

1. Packet arrives from Layer 3 with IP address

2. Layer 2 wraps it in a frame

3. Source MAC + Destination MAC added

4. Switch reads destination MAC

5. Frame forwarded to correct port
